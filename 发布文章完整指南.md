# 📝 NovaTech 博客 - 发布文章完整指南

## 🎯 发布流程概述

发布新文章只需要 **3 个简单步骤**：

1. **登录管理员账户** → 2. **进入 Write 编辑页** → 3. **填写内容并发布**

---

## 📌 完整步骤详解

### **第 1 步：导航到登录页面**

#### 方式 A：通过导航栏
```
1. 打开博客首页 (http://localhost:5173)
2. 点击右上角导航栏的 "Write" 按钮
3. 自动跳转到登录页面
```

#### 方式 B：直接访问
```
http://localhost:5173/admin/login
```

---

### **第 2 步：登录管理员账户**

在登录页面输入以下凭证：

| 字段 | 值 |
|------|-----|
| **Username（用户名）** | `admin` |
| **Password（密码）** | `password123` |

```
⚠️ 注意：
- 用户名和密码都要准确输入
- 密码区分大小写
- 确保后端服务正在运行 (http://localhost:8080)
```

**登录时可能出现的问题：**

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| "Invalid username or password" | 凭证错误 | 确认用户名是 `admin`，密码是 `password123` |
| "Login failed. Please try again." | 后端连接失败 | 检查后端是否运行在 8080 端口 |
| 页面卡住 | 网络问题 | 打开浏览器控制台查看错误信息 |

---

### **第 3 步：进入编辑页面**

登录成功后，会自动跳转到 `/admin/editor` 页面。

你会看到：
- ✅ 左侧：**内容输入区**
- ✅ 右侧：**实时预览区**（桌面端）
- ✅ 顶部：**工具栏**（标题、状态、发布按钮）

---

## ✍️ 编写文章

### **必填字段**

#### **1. Post Title（文章标题）**
```
位置: 编辑区最上方
类型: 文本输入
示例: "Spring Boot 微服务架构设计"
提示: 标题会显示在左侧编辑区，也会在预览区显示
```

#### **2. Content（文章内容）**
```
位置: 编辑区下方大文本框
类型: HTML 或 Markdown 文本
说明: 
  - 输入的内容会实时显示在右侧预览区
  - 支持 HTML 格式（推荐）
  - 格式化示例见下方
```

### **可选字段**

#### **3. Brief excerpt（文章摘要）**
```
位置: 标题下方
类型: 文本输入
作用: 文章卡片上显示的简短描述
自动: 如果留空，会自动取内容的前 100 个字符
```

#### **4. Category（分类）**
```
位置: 编辑区中部
类型: 下拉菜单
可选值:
  - Frontend (前端)
  - Backend (后端)
  - AI (人工智能)
  - DevOps (运维/部署)
  - General (通用)
默认: General
```

#### **5. Tags（标签）**
```
位置: 编辑区中部，分类右侧
类型: 逗号分隔的文本
格式: "标签1, 标签2, 标签3"
示例: "Spring Boot, Java, Microservices, Architecture"
作用: 帮助文章分类和搜索
```

#### **6. Read time（阅读时间）**
```
位置: 编辑区中部，标签右侧
类型: 文本输入
格式: "X min" 或 "X minute"
示例: "5 min" 或 "10 minute"
默认: "5 min"
```

## 🚀 发布文章

### **发布按钮**

```
位置: 顶部工具栏右侧
按钮文本: "Publish"（或 "Publishing..." 发布中）
颜色: 青色背景
```

### **发布前检查清单**

发布前确保：

- ✅ **标题已填写**（必需）
- ✅ **内容已填写**（必需）
- ✅ **分类已选择**（可选，默认 General）
- ✅ **标签已添加**（可选）
- ✅ **阅读时间已设置**（可选，默认 5 min）
- ✅ **预览内容看起来正确**（右侧预览区）

### **发布流程**

```
1. 确认所有必填字段已完成
2. 点击 "Publish" 按钮
3. 按钮变为 "Publishing..." （显示加载状态）
4. 等待 2-5 秒
5. 自动跳转回首页
6. 新文章应该出现在文章列表的最顶部
```

## 🐛 发布失败常见问题

### **问题 1：点击 Publish 但没有反应**

**原因：** 
- 标题或内容为空
- 后端服务未运行
- JWT 令牌过期

**解决方案：**
```
1. 检查标题和内容是否都有输入
2. 检查后端是否运行 (http://localhost:8080)
3. 刷新页面重新登录
4. 查看浏览器控制台（F12 → Console）是否有错误信息
```

### **问题 2：发布时出现错误消息**

**常见错误信息及解决方案：**

| 错误信息 | 原因 | 解决方案 |
|---------|------|--------|
| "Not authenticated - please login first" | 登录过期 | 重新登录 (刷新页面) |
| "Failed to save post" | 后端错误 | 检查后端日志 |
| "Network error" | 网络连接失败 | 检查后端地址是否为 http://localhost:8080 |

**查看详细错误信息：**
```
1. 打开浏览器开发者工具 (F12)
2. 切换到 "Console" 标签
3. 查看红色错误信息
4. 查看 "Network" 标签检查 API 请求状态
```

### **问题 3：发布后文章没有出现在首页**

**原因：**
- 页面缓存
- API 请求成功但数据未刷新

**解决方案：**
```
1. 等待 2-3 秒让页面完全加载
2. 手动刷新页面 (F5 或 Ctrl+R)
3. 在浏览器打开 http://localhost:8080/api/posts 查看所有文章
4. 查看新文章是否确实被创建
```

## 💡 编写建议

### **✅ 最佳实践**

1. **标题清晰明确**
   - 使用动词：构建、实现、优化
   - 例：❌ "React" → ✅ "在 React 中实现高性能列表渲染"

2. **摘要简洁**
   - 40-100 字之间最佳
   - 概括主要内容，吸引读者

3. **内容结构清晰**
   - 使用 `<h2>` 划分章节
   - 使用列表组织要点
   - 使用引用框突出重要观点

4. **标签准确**
   - 3-5 个标签最佳
   - 包含技术栈和主题
   - 便于分类和搜索

5. **代码示例完整**
   - 提供可运行的示例
   - 包含详细的注释
   - 说明如何使用

### **❌ 常见错误**

- ❌ 标题过长或过于通用
- ❌ 摘要为空或过长
- ❌ 没有段落划分，全是一大块文本
- ❌ 标签过多或不相关
- ❌ 图片或代码块格式混乱

目前编辑器**不支持直接编辑已发布的文章**。

```

### **个人信息安全**

```
⚠️ 重要注意：

```

```
